§ 01 — the cache timer

Know when to act.

A line inside Claude Code's native status area that tells you exactly where your prompt cache stands. Reads the session JSONL, computes state from message.usage fields, renders one authoritative line every second.

◉cache warm·5m·4:32 left·Opus 4.7 plenty of TTL
◉cache warm·5m·0:58 left·Opus 4.7 under 25% — running low
○cache cold·12m past·/compress recommended time to trim
◉cache active·agent working·Opus 4.7 mid-turn (tool_use)

Derived from fields Claude Code already writes — no proxying, no wire-level interception. Per-session mtime cache so idle ticks skip the JSONL parse entirely. Client-side commands like /context and /clear are filtered so they don't lock the display into “working”.

§ 02 — /compress

Trim without leaving.

A slash command wired through UserPromptSubmit. Type /compress and the active session's JSONL gets a trimmed sibling with a fresh UUID. Six modes — Redact, Recency N, Focus N, Smart, Ultra, Truncate N. Original file untouched.

┌─ claudecompress ────────────────────────────────────────┐ mode: focus (last 20) · dropped thinking tokens: 628.8k → 126.0k (saved ≈ 502.8k) cold $ $9.43 → $1.89 (saved ≈ $7.54) [Opus 4.7] trimmed: 17420d99-7152-4359-bfdd-34c2cefe77e3 └─────────────────────────────────────────────────────────┘ Ctrl+C, then: claude --resume 17420d99…

Want it seamless? ccw wraps claude and auto-resumes into the trimmed session when you Ctrl+C — flags like --dangerously-skip-permissions preserved across the relaunch.

§ 03 — the retrospective trimmer

One command, any session.

For sessions you didn't trim in the moment — the interactive picker lists every project, flags cache staleness, shows estimated cold-/resume cost, writes a new JSONL alongside the old one.

$ bunx claudecompress # auto-detects the current project # shows each session's size, staleness, cost estimate # pick a mode, confirm — done

§ 05 — install

Two commands, one restart.

Global install is required for the status line — refreshInterval: 1 polling needs fast startup, and npx's cold-start would eat CPU. Bun is preferred over node (~3× faster startup → smoother ticking). The /compress hook itself is fine via npx.

# preferred $ bun add -g claudecompress # or $ npm i -g claudecompress # wire up hook + status line in ~/.claude/settings.json $ claudecompress install # fully quit and relaunch Claude Code. done.

The installer backs up your previous settings to settings.json.claudecompress.bak and asks before touching anything. Existing custom statusLine entries are never overwritten without explicit confirm. Subcommands — install-hook, install-statusline, history, uninstall — for surgical edits later.
